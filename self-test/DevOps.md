# DevOps

## 1. DevOps 정의

DevOps는 Development와 Operations 결합한 소프트웨어 개발 방법론으로, 개발과 운영 간 협업 강화, CI/CD 통한 자동화, 지속적 피드백으로 빠른 배포 주기와 안정적 서비스 운영 실현하는 문화 및 실천 방법론

## 2. DevOps 설명

### 1) DevOps CI/CD 프로세스 (그림)

```
┌──────────────────────────────────────────────────────────────────┐
│                         DevOps Lifecycle                          │
└──────────────────────────────────────────────────────────────────┘

        [Plan]           [Code]          [Build]         [Test]
          │                │               │               │
          ↓                ↓               ↓               ↓
    요구사항 분석  ───→  코드 작성  ───→  빌드 자동화  ───→  자동화 테스트
          │                │               │               │
       Jira, Trello    Git, GitHub      Maven,         JUnit,
                                        Gradle         Selenium
                                                            │
                                                            ↓
┌──────────────────────────────────────────────────────────────────┐
│                         Continuous Feedback                       │
└──────────────────────────────────────────────────────────────────┘
                                                            │
        [Monitor]        [Operate]       [Deploy]          │
          │                │               │               │
          ↓                ↓               ↓               ↓
    성능 모니터링  ←───  서비스 운영  ←───  자동화 배포  ←───┘
          │                │               │
     Prometheus,      Kubernetes,       Jenkins,
     Grafana          Docker            GitLab CI
```

- 간글: DevOps 프로세스는 계획-개발-빌드-테스트-배포-운영-모니터링 단계로 구성되며, 각 단계마다 자동화 도구 활용하여 지속적 통합과 전달 실현. 모니터링 결과는 다시 계획 단계로 피드백되어 지속적 개선 순환 형성

### 2) DevOps 구성 요소 (표)

| 구분 | 세부 항목 | 설명 |
|------|-----------|------|
| 개발 영역 | 버전 관리 | Git, GitHub, GitLab 통한 소스 코드 관리 |
| | 빌드 자동화 | Maven, Gradle 통한 컴파일 및 패키징 |
| | 코드 품질 | SonarQube, Checkstyle 통한 정적 분석 |
| 테스트 영역 | 단위 테스트 | JUnit, pytest 통한 코드 단위 검증 |
| | 통합 테스트 | Selenium, Postman 통한 시스템 통합 검증 |
| | 성능 테스트 | JMeter, Gatling 통한 부하 및 성능 검증 |
| 배포 영역 | CI/CD | Jenkins, GitLab CI, GitHub Actions 통한 자동화 파이프라인 |
| | 컨테이너화 | Docker 통한 애플리케이션 패키징 |
| | 오케스트레이션 | Kubernetes 통한 컨테이너 배포 및 관리 |
| 운영 영역 | 인프라 코드화 | Terraform, Ansible 통한 인프라 자동화 |
| | 모니터링 | Prometheus, Grafana, ELK Stack 통한 성능 추적 |
| | 로그 관리 | Fluentd, Logstash 통한 로그 수집 및 분석 |
| 문화적 요소 | 협업 문화 | 개발-운영 팀 간 벽 제거, 공동 책임 |
| | 자동화 우선 | 반복 작업 자동화로 인적 오류 최소화 |
| | 지속적 개선 | 빠른 피드백 기반 지속적 학습 및 개선 |

- 간글: DevOps는 개발, 테스트, 배포, 운영 영역의 기술 스택과 협업 문화를 포괄하는 종합 방법론. 각 영역에서 자동화 도구 활용하여 빠른 배포 주기, 높은 코드 품질, 안정적 운영 동시에 달성하며, 개발과 운영 간 긴밀한 협업 통해 비즈니스 가치 신속히 전달
