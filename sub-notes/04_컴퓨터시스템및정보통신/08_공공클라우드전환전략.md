# 공공 클라우드 전환 전략

## 1. 공공 클라우드 전환 정의

행정안전부 2025년까지 19,000개 공공 시스템을 클라우드로 전환하는 Cloud-First 정책으로, 레거시 시스템을 클라우드 네이티브 아키텍처로 전환하여 인프라 비용 절감, 서비스 확장성 향상, 디지털 전환 가속화를 달성하는 전략

## 2. 공공 클라우드 전환 설명

### 1) 6R 마이그레이션 전략 (그림)

```
┌─────────────────────────────────────────────────────────┐
│              레거시 시스템 평가                          │
└─────────────┬───────────────────────────────────────────┘
              │
    ┌─────────┴─────────┐
    │  마이그레이션 전략  │
    └─────────┬─────────┘
              │
    ┌─────────┼──────────────────────────────────┐
    ↓         ↓         ↓         ↓         ↓    ↓
┌────────┐┌────────┐┌────────┐┌────────┐┌────┐┌────────┐
│Rehost  ││Replatform│Refactor││Repurchase│Retire│Retain  │
│(Lift&  ││(Lift&  ││(Re-    ││(SaaS   ││    ││(유지)  │
│Shift)  ││Reshape)││architect│Replace) ││    ││        │
└────────┘└────────┘└────────┘└────────┘└────┘└────────┘
   ↓         ↓         ↓         ↓       ↓      ↓
[IaaS]   [PaaS]   [Container] [SaaS]  [폐기] [온프레미스]
 40%      20%       15%        10%     10%     5%
```

- 간글: Gartner 6R 전략 기반, 시스템 특성에 따라 적절한 전환 방식 선택. Rehost 빠른 전환, Refactor 최대 효과, Retain 점진적 전환 통해 위험 최소화

### 2) 단계별 전환 로드맵 (표)

| 구분 | 세부 항목 | 설명 |
|------|-----------|------|
| **1단계<br>평가** | 자산 목록 작성 | 19,000개 시스템 현황 파악 (기능, 의존성, 데이터) |
| | 클라우드 적합성 평가 | 6R 전략 매핑, TCO 분석 |
| | 보안 요구사항 | CSAP 인증 대상 식별, ISMS-P 준수 |
| **2단계<br>파일럿** | 비핵심 시스템 우선 | 정적 웹사이트, 개발/테스트 환경 |
| | PoC 수행 | AWS/Azure/GCP 3개월 테스트 |
| | 조직 역량 강화 | 클라우드 아키텍트 교육, 보안 교육 |
| **3단계<br>전환** | 핵심 시스템 이관 | 민원24, 정부24 등 대국민 서비스 |
| | 하이브리드 운영 | 온프레미스 + 클라우드 병행 6개월 |
| | 데이터 마이그레이션 | AWS DMS, Azure Data Factory 활용 |
| **4단계<br>최적화** | 비용 최적화 | RI/SP 구매, 오토스케일링 |
| | 성능 튜닝 | CDN, 캐싱, DB 최적화 |
| | 거버넌스 확립 | FinOps, 태깅 전략, 비용 모니터링 |

- 간글: 총 15개월 소요 예상. 단계적 접근으로 대규모 장애 방지, 파일럿 통해 검증 후 전면 확대. 하이브리드 운영 기간 중 안정성 확보

### 3) CSAP 보안 인증 요구사항 (표)

| 구분 | 세부 항목 | 설명 |
|------|-----------|------|
| **인증 대상** | 공공 클라우드 서비스 | AWS, Azure, GCP, NCP 등 |
| | 클라우드 이용 기관 | 중앙부처, 지자체, 공공기관 |
| **보안 영역** | 접근 통제 | MFA, RBAC, 네트워크 ACL |
| | 암호화 | 전송 중(TLS 1.3), 저장 중(AES-256) |
| | 로그 관리 | CloudTrail, 6개월 보관 |
| | 취약점 관리 | 월 1회 스캔, 즉시 패치 |
| **컴플라이언스** | 개인정보보호법 | 개인정보 국내 저장 원칙 |
| | 전자정부법 | 클라우드 이용 승인 절차 |
| | ISMS-P | 정보보호 및 개인정보보호 통합 인증 |

- 간글: CSAP 미인증 클라우드 사용 불가. 기존 ISMS-P 대비 클라우드 특화 항목 추가(공동 책임 모델, 멀티테넌시 보안). 인증 획득 평균 6개월 소요

## 3. 전환 효과 및 고려사항

### 1) 기대 효과
- **비용 절감**: 인프라 운영비 40% 감소 (서버 구매, 전력, 인건비)
- **확장성**: 트래픽 급증 시 Auto Scaling (민원24 접속 폭주 대응)
- **재해복구**: RTO 24시간 → 1시간, Multi-Region 배포

### 2) 주요 과제
- **보안 우려**: 데이터 주권, 클라우드 보안 사고 (2023년 NCP 장애)
- **종속성 위험**: 특정 CSP Lock-in, 멀티클라우드 복잡도
- **레거시 호환**: 30년된 COBOL 시스템 리팩토링 비용
- **조직 저항**: 공무원 클라우드 역량 부족, 변화 관리 필요

### 3) 해결 방안
- **하이브리드 전략**: 민감 데이터 온프레미스, 나머지 클라우드
- **컨테이너화**: Docker/Kubernetes로 CSP 독립성 확보
- **단계적 교육**: AWS/Azure 자격증 취득 지원, 핸즈온 랩
