# 답안 구조 템플릿

## 기본 구조

```
1. [토픽명] 정의
   - [토픽명]의 특징/목적/기술을 포함한 명확한 정의 (명사형)

2. [토픽명] 설명
   1) [토픽명]의 [구조/프로세스/아키텍처/흐름] (그림)

      [구성요소 간 관계를 보여주는 다이어그램]

      - 간글: [그림에 대한 2-3문장 설명]

   2) [토픽명]의 [분류/유형/특징/구성요소] (표)

      | 구분 | 세부 항목 | 설명 |
      |------|-----------|------|
      | [카테고리1] | [항목1-1] | [설명1-1] |
      | | [항목1-2] | [설명1-2] |
      | [카테고리2] | [항목2-1] | [설명2-1] |
      | | [항목2-2] | [설명2-2] |
      | [카테고리3] | [항목3-1] | [설명3-1] |
      | | [항목3-2] | [설명3-2] |

      - 간글: [표에 대한 2-3문장 설명]
```

## 그림 표현 예시

### 프로세스 흐름형
```
[단계1: 계획] → [단계2: 개발] → [단계3: 테스트] → [단계4: 배포]
     ↓              ↓              ↓              ↓
  [도구/기술]    [도구/기술]    [도구/기술]    [도구/기술]
```

### 아키텍처형
```
┌─────────────────────────────────────┐
│          [상위 레이어]               │
├─────────────────────────────────────┤
│  [중간 레이어1]  │  [중간 레이어2]   │
├─────────────────────────────────────┤
│          [하위 레이어]               │
└─────────────────────────────────────┘
```

### 순환형
```
        [단계1]
           ↓
[단계4] ← [단계2]
   ↑         ↓
      [단계3]
```

### 계층형
```
사용자 인터페이스
        ↓
   비즈니스 로직
        ↓
   데이터 액세스
        ↓
     데이터베이스
```

## 표 작성 예시

### 분류형
```
| 구분 | 세부 항목 | 설명 |
|------|-----------|------|
| 개발 단계 | 계획 | 요구사항 분석, 설계 |
| | 구현 | 코드 작성, 단위 테스트 |
| 운영 단계 | 배포 | 자동화 배포, 모니터링 |
| | 유지보수 | 장애 대응, 성능 최적화 |
| 지원 도구 | CI/CD | Jenkins, GitLab CI |
| | 모니터링 | Prometheus, Grafana |
```

### 특징형
```
| 구분 | 세부 항목 | 설명 |
|------|-----------|------|
| 기술적 특징 | 자동화 | 빌드, 테스트, 배포 자동화 |
| | 통합 | 지속적 통합 및 전달 |
| 조직적 특징 | 협업 | 개발-운영 팀 협업 |
| | 문화 | 빠른 피드백, 지속적 개선 |
| 비즈니스 특징 | 속도 | 빠른 출시 주기 |
| | 품질 | 안정적 서비스 운영 |
```

### 구성요소형
```
| 구분 | 세부 항목 | 설명 |
|------|-----------|------|
| 개발 도구 | 버전 관리 | Git, GitHub |
| | 빌드 도구 | Maven, Gradle |
| 테스트 도구 | 단위 테스트 | JUnit, pytest |
| | 통합 테스트 | Selenium, Postman |
| 운영 도구 | 컨테이너 | Docker, Kubernetes |
| | 모니터링 | Prometheus, ELK Stack |
```

## 간글 작성 예시

**그림 간글 예시:**
- "[토픽명] 프로세스는 [단계1]에서 [단계N]까지 순차적으로 진행. 각 단계에서 [도구/기술] 활용하여 [목적] 달성."

**표 간글 예시:**
- "[토픽명]은 [카테고리1], [카테고리2], [카테고리3]으로 구성. 각 영역에서 [핵심 키워드] 중심으로 [목적] 실현."

## 작성 시 체크리스트

- [ ] 정의에 특징/목적/기술 포함
- [ ] 정의 마무리가 명사형
- [ ] 목차만 봐도 내용 파악 가능
- [ ] 그림에 구성요소 간 관계 표현
- [ ] 표가 3단표 형식
- [ ] 표에 그룹핑이 명확함
- [ ] 그림 아래 간글 작성
- [ ] 표 아래 간글 작성
- [ ] 조사 생략
- [ ] 1페이지 분량
