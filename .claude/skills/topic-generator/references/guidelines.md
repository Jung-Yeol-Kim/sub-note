# 답안 작성 가이드

## 기본 원칙

1. **정의 작성**
   - 특징, 목적, 기술이 포함된 명확한 구성
   - 마무리 단어는 명사형 (예: 애자일 방법론, 지속적 통합 프로세스)
   - 2-3문장으로 간결하게 정리

2. **목차 구성**
   - 문제에 집중하여 1단락, 2단락으로 구성
   - 목차만 봐도 내용을 알 수 있도록 작성
   - 각 항목은 명확한 제목으로 구분

3. **그림 작성 (2.1)**
   - 구성요소 사이의 관계, 활동, 기술이 보이도록 작성
   - 균형있고 깔끔한 다이어그램
   - ASCII 아트나 마크다운 표현 활용
   - 화살표로 흐름 표시

4. **표 작성 (2.2)**
   - 3단표 (구분 | 세부 항목 | 설명) 형식
   - 그룹핑에 신경써서 구성
   - 키워드가 최대한 명확하게 보이도록 작성
   - 논리적 순서로 배치

5. **간글 작성**
   - 2단락 1)그림과 2)표 아래에 각각 작성
   - 2-3문장으로 핵심 내용 요약
   - 조사 생략, 간결하고 명확한 표현

6. **분량 조절**
   - 1페이지로 압축하여 작성
   - 불필요한 내용 제거
   - 핵심 키워드 중심으로 구성

7. **작성 스타일**
   - 조사 생략 (예: "DevOps는" → "DevOps")
   - 간결하고 명확한 표현
   - 전문 용어는 정확하게 사용
   - 불필요한 수식어 제거

## 작성 프로세스

1. **주제 분석**
   - 문제에서 요구하는 핵심 내용 파악
   - 관련 키워드 및 개념 정리

2. **정의 작성**
   - 특징/목적/기술 포함
   - 명사형으로 마무리

3. **목차 구성**
   - 1단락: 정의
   - 2단락: 설명 (1) 그림, 2) 표)

4. **그림 작성**
   - 구성요소 식별
   - 관계 및 흐름 표현

5. **표 작성**
   - 3단표 구조 설정
   - 그룹핑 및 키워드 배치

6. **간글 작성**
   - 그림과 표에 대한 간략한 설명

7. **검토 및 수정**
   - 1페이지 분량 확인
   - 조사 제거
   - 명확성 점검

## 주의사항

- 문제에서 요구하는 내용에 집중
- 불필요한 배경 설명 최소화
- 핵심 키워드 중심으로 작성
- 그림과 표의 균형 유지
- 일관된 표현 스타일 유지
