# 평가 기준 상세 설명

## 6대 평가 기준

정보관리기술사 답안은 다음 6가지 기준으로 평가됩니다. 각 기준당 5점 만점, 총 30점 만점입니다.

---

## 1. 첫인상 (First Impression) - 5점

### 평가 내용
답안의 구조적, 시각적 완성도를 평가합니다. 채점자가 답안을 처음 봤을 때 받는 인상이 중요합니다.

### 세부 평가 항목

#### 1.1 구조적 완성도 (2점)
- **서론-본론-결론 구조** 명확한가?
- **목차 구성** 논리적이고 명확한가?
- **단락 구분** 적절한가?

**점수 기준:**
- 2점: 서론-본론-결론 명확, 목차 논리적
- 1.5점: 구조는 있으나 일부 불명확
- 1점: 구조가 약하거나 불완전
- 0.5점: 구조가 매우 불명확

#### 1.2 시각적 완성도 (2점)
- **다이어그램과 표** 균형있게 배치되었는가?
- **레이아웃** 깔끔하고 정돈되었는가?
- **가독성** 좋은가? (글씨 크기, 직선, 정렬)

**점수 기준:**
- 2점: 다이어그램+표 모두 있고 깔끔함
- 1.5점: 둘 다 있으나 일부 조잡함
- 1점: 하나만 있거나 둘 다 조잡함
- 0.5점: 둘 다 없거나 매우 조잡함

#### 1.3 분량 (1점)
- **적절한 분량**인가? (1페이지 원칙)
- 너무 빈약하거나 과도하지 않은가?

**점수 기준:**
- 1점: 1페이지 분량 적절
- 0.7점: 약간 부족하거나 과다
- 0.5점: 분량 부족 또는 과다

### 체크리스트
- [ ] 서론-본론-결론 구조 명확
- [ ] 다이어그램 포함
- [ ] 표 포함
- [ ] 깔끔한 레이아웃
- [ ] 적절한 분량

## 2. 출제반영성 (Question Reflection) - 5점

### 평가 내용
출제자의 의도를 정확히 파악하고 요구사항을 충족했는지 평가합니다.

### 세부 평가 항목

#### 2.1 문제 의도 파악 (2점)
- 문제 유형을 정확히 파악했는가? (정의형/설명형/비교형/절차형/분석형)
- 핵심 키워드를 다루었는가?

**점수 기준:**
- 2점: 문제 유형 정확히 파악, 핵심 키워드 모두 다룸
- 1.5점: 대체로 파악했으나 일부 누락
- 1점: 파악 부족, 핵심 키워드 일부만
- 0.5점: 문제 의도 크게 벗어남

#### 2.2 요구사항 충족 (2점)
- 문제에서 요구한 내용을 모두 다루었는가?
- 적절한 깊이로 설명했는가?

**점수 기준:**
- 2점: 요구사항 모두 충족, 깊이 적절
- 1.5점: 대부분 충족, 깊이 약간 부족
- 1점: 일부만 충족
- 0.5점: 요구사항 대부분 미충족

#### 2.3 범위 준수 (1점)
- 문제 범위를 벗어나지 않았는가?
- 불필요한 내용으로 지면 낭비하지 않았는가?

**점수 기준:**
- 1점: 범위 적절
- 0.5점: 일부 벗어남
- 0점: 크게 벗어남

### 문제 유형별 체크리스트

**정의형 ("~이란?", "정의")**
- [ ] 특징, 목적, 기술 포함된 정의
- [ ] 명사형 종결
- [ ] 명확하고 간결한 설명

**설명형 ("설명하시오")**
- [ ] 다이어그램으로 구조/프로세스 표현
- [ ] 표로 분류/특징 정리
- [ ] 간글로 핵심 설명

**비교형 ("비교", "차이점")**
- [ ] 두 대상 모두 명확히 설명
- [ ] 비교 항목별 대조
- [ ] 장단점 및 선택 기준 제시

**절차형 ("프로세스", "절차")**
- [ ] 단계별 흐름도
- [ ] 각 단계별 활동 및 산출물
- [ ] 순서 논리적

**분석형 ("분석", "평가")**
- [ ] 현황 분석
- [ ] 문제점 도출
- [ ] 해결방안 제시

## 3. 논리성 (Logical Consistency) - 5점

### 평가 내용
키워드의 계층적 배치와 논리적 흐름을 평가합니다.

### 세부 평가 항목

#### 3.1 계층적 구조 (2점)
- 대분류 → 중분류 → 소분류 계층 명확한가?
- 키워드가 논리적으로 그룹핑되었는가?

**점수 기준:**
- 2점: 계층 구조 명확, 그룹핑 논리적
- 1.5점: 계층 구조 있으나 일부 불명확
- 1점: 계층 구조 약함
- 0.5점: 계층 구조 없음

#### 3.2 논리적 흐름 (2점)
- 서론에서 결론까지 자연스럽게 연결되는가?
- 단락 간 연결이 매끄러운가?

**점수 기준:**
- 2점: 흐름 자연스럽고 매끄러움
- 1.5점: 대체로 자연스러우나 일부 끊김
- 1점: 연결이 부자연스러움
- 0.5점: 논리적 흐름 부족

#### 3.3 일관성 (1점)
- 용어 사용이 일관적인가?
- 관점이 일관되는가?

**점수 기준:**
- 1점: 용어와 관점 일관적
- 0.5점: 일부 불일치
- 0점: 일관성 없음

### 체크리스트
- [ ] 대제목 4개 내외
- [ ] 계층적 키워드 배치
- [ ] 단락 간 연결 텍스트
- [ ] 용어 일관성
- [ ] 서론→본론→결론 자연스러운 흐름

## 4. 응용능력 (Application Ability) - 5점

### 평가 내용
단순 암기를 넘어 이해를 바탕으로 한 설명/설득 능력을 평가합니다.

### 세부 평가 항목

#### 4.1 이해 기반 설명 (2점)
- 단순 나열이 아닌 설명인가?
- "왜" 필요한지, "어떻게" 동작하는지 명확한가?

**점수 기준:**
- 2점: 원리와 동작 방식 명확히 설명
- 1.5점: 설명 있으나 깊이 부족
- 1점: 설명 부족, 나열 위주
- 0.5점: 거의 나열만

#### 4.2 실무 연계 (2점)
- 실제 활용 사례를 제시했는가?
- 구현 경험이나 실무 관점이 담겨 있는가?

**점수 기준:**
- 2점: 구체적 사례와 실무 관점 명확
- 1.5점: 사례 있으나 추상적
- 1점: 사례 부족
- 0.5점: 실무 연계 거의 없음

#### 4.3 설득력 (1점)
- 타인에게 설명할 수 있는 수준인가?
- 논리적으로 설득력 있는가?

**점수 기준:**
- 1점: 매우 설득력 있음
- 0.5점: 보통
- 0점: 설득력 부족

### 좋은 응용 예시

**나쁜 예 (단순 암기):**
```
CI/CD 도구
- Jenkins: 자동화 도구
- GitLab CI: CI/CD 플랫폼
- GitHub Actions: 워크플로우 자동화
```

**좋은 예 (응용/설명):**
```
CI/CD 파이프라인 구축 도구, 개발부터 배포까지 자동화

| 구분 | 도구 | 역할 및 활용 |
|------|------|--------------|
| 통합/배포 | Jenkins | 플러그인 기반 확장성, 기존 시스템 통합 용이. 코드 커밋 시 자동 빌드/테스트 |
| 통합 플랫폼 | GitLab CI | Git 저장소 통합, .gitlab-ci.yml로 파이프라인 정의. Docker 기반 실행 환경 |
| 클라우드 네이티브 | GitHub Actions | 이벤트 기반 워크플로우, 마켓플레이스 액션 재사용. 클라우드 환경 최적화 |

- 간글: 각 도구, 조직 환경과 기술 스택에 따라 선택.
  온프레미스 환경 Jenkins, GitLab 중심 개발 GitLab CI,
  오픈소스 프로젝트 GitHub Actions 활용.
  자동화로 배포 주기 단축, 품질 일관성 확보.
```

## 5. 특화 (Specialization) - 5점

### 평가 내용
전문가적 관점과 전문성 수준을 평가합니다.

### 세부 평가 항목

#### 5.1 전문 용어 정확성 (2점)
- 기술 용어를 정확하게 사용했는가?
- 약어와 정식 명칭을 적절히 사용했는가?

**점수 기준:**
- 2점: 전문 용어 정확, 적절한 수준
- 1.5점: 대체로 정확하나 일부 부정확
- 1점: 용어 사용 부정확 또는 부족
- 0.5점: 전문성 낮음

#### 5.2 깊이 (2점)
- 표면적 설명이 아닌 심층적 이해를 보여주는가?
- 기술의 원리와 작동 방식을 설명하는가?

**점수 기준:**
- 2점: 깊이 있는 설명, 원리 명확
- 1.5점: 적절한 깊이
- 1점: 표면적 설명
- 0.5점: 매우 얕은 설명

#### 5.3 최신성 (1점)
- 최신 기술 트렌드를 반영했는가?
- 현재 산업에서 사용되는 내용인가?

**점수 기준:**
- 1점: 최신 트렌드 잘 반영
- 0.5점: 보통 수준
- 0점: 구식 내용

### 전문성 향상 요소

**일반적 답안 vs 전문가 답안:**

| 측면 | 일반적 | 전문가 |
|------|--------|--------|
| 용어 | "데이터 복제" | "Active-Active Replication, Async Replication" |
| 수치 | "성능 좋음" | "처리량 3배 향상, 지연시간 100ms → 30ms" |
| 사례 | "많이 사용" | "Netflix, Kafka로 일일 수조 건 이벤트 처리" |
| 기술 | "최신 기술" | "CQRS 패턴, Event Sourcing 기반 설계" |

### 체크리스트
- [ ] 정확한 기술 용어 사용
- [ ] 원리/작동 방식 설명
- [ ] 구체적 수치/사례
- [ ] 최신 트렌드 반영
- [ ] 표준/아키텍처 패턴 언급

## 6. 견해 (Perspective) - 5점

### 평가 내용
통찰력 있는 의견과 우수한 견해를 평가합니다.

### 세부 평가 항목

#### 6.1 독창성 (2점)
- 남들과 다른 관점을 제시했는가?
- 새로운 시각을 보여주는가?

**점수 기준:**
- 2점: 독창적이고 신선한 관점
- 1.5점: 일부 독창성 있음
- 1점: 일반적인 관점
- 0.5점: 독창성 없음

#### 6.2 비판적 사고 (2점)
- 장단점을 균형있게 분석했는가?
- 한계점을 지적하고 개선 방안을 제시했는가?

**점수 기준:**
- 2점: 비판적 분석 우수, 개선안 구체적
- 1.5점: 장단점 분석 있음
- 1점: 일방적 서술
- 0.5점: 비판적 사고 부족

#### 6.3 미래 전망 (1점)
- 발전 방향을 제시했는가?
- 융합 기술이나 트렌드를 언급했는가?

**점수 기준:**
- 1점: 구체적이고 통찰력 있는 전망
- 0.5점: 추상적인 전망
- 0점: 전망 없음

### 견해 제시 영역

1. **서론**: 기술의 중요성, 필요성에 대한 견해
2. **본론**: 특정 기술 선택 이유, 장단점 분석
3. **결론**: 발전 방향, 해결 방안, 통찰

### 견해 유형

**융합형:**
"Kubernetes와 서버리스 기술 결합, FaaS on Kubernetes 형태로 진화.
컨테이너 오케스트레이션의 유연성과 서버리스의 편의성 동시 확보."

**문제 해결형:**
"마이크로서비스 아키텍처, 서비스 간 통신 복잡도 증가 문제.
Service Mesh 패턴으로 트래픽 관리, 보안, 모니터링 중앙화.
Istio, Linkerd 등 활용하여 복잡도 감소."

**트렌드 예측형:**
"AI/ML 워크로드, GPU 자원 효율적 활용 위해 컨테이너화 가속.
MLOps 플랫폼과 통합, 모델 학습부터 배포까지 자동화 파이프라인 구축.
2025년 이후 AI 네이티브 인프라로 진화 전망."

**비교 분석형:**
"모놀리식 vs 마이크로서비스, 조직 규모와 성숙도에 따라 선택.
소규모 팀 모놀리식으로 빠른 출시, 대규모 조직 마이크로서비스로 독립 배포.
단계적 전환 전략 Strangler Fig 패턴 활용 권장."

### 체크리스트
- [ ] 독창적 관점 제시
- [ ] 장단점 균형 분석
- [ ] 한계점 및 개선안
- [ ] 구체적 미래 전망
- [ ] 융합/발전 방향 제시

## 종합 평가 가이드

### 점수대별 수준

**25-30점: 우수**
- 모든 기준 충족
- 구조, 내용, 전문성 모두 뛰어남
- 독창적 견해 포함
- 실무 경험 잘 녹아있음

**18-24점: 합격 수준**
- 기본 구조 갖춤
- 요구사항 충족
- 일부 개선 필요
- 전문성은 있으나 깊이 부족

**12-17점: 보완 필요**
- 구조 불완전
- 요구사항 일부만 충족
- 전문성 부족
- 많은 개선 필요

**0-11점: 불합격**
- 기본 구조 미흡
- 요구사항 미충족
- 내용 부실
- 전면 재작성 필요

### 채점 시 주의사항

1. **객관성 유지**: 개인 선호가 아닌 기준에 따라 평가
2. **증거 기반**: 구체적 근거를 들어 점수 부여
3. **일관성**: 모든 답안에 동일한 기준 적용
4. **건설적 피드백**: 개선 방향 명확히 제시
5. **균형**: 긍정적 측면과 개선 측면 균형있게 평가
